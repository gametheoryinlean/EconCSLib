# Staged Knowledge Topic Catalog

This catalog records the immediate staged topic folders. The frontmatter is a
compatibility header for mdblueprint versions before exact `topics.md` files are
reserved.

## Topics

- `approachability` - approachability theorem statements and proof routes.
- `cooperative_game` - transferable-utility coalitional games, the core, balancedness, convex games, and Shapley value.
- `foundation` - shared vocabulary (preference, profile shim, argmax) consumed by every other topic.
- `extensive_game` - extensive-form game concepts and finite-tree infrastructure.
- `fixed_point` - fixed-point theorem targets and reductions.
- `lattice` - lattice support facts.
- `math.simplex` - `stdSimplex` utilities (weighted sums, point masses, bounds, continuity).
- `math.linear_algebra` - linear algebra support facts.
- `math.linear_programming` - LP duality and complementarity support.
- `mechanism_design` - mechanism-design primitives and theorem interfaces.
- `order` - order support facts.
- `strategic_game` - normal-form game concepts, theorem targets, correlated equilibrium, learning, Bayesian games, dynamics, and evolution.
- `repeated_game` - repeated games and folk-theorem targets.
- `stochastic_game` - stochastic-game definitions, values, and examples.
- `utility` - preferences, lotteries, and expected-utility material.
- `zero_sum` - zero-sum games, minimax, and value theory.

## Boundary

Place a new staged node in the folder for its main mathematical ownership. Use
tags or additional topic membership for cross-cutting concepts rather than
duplicating node files.
